## 💡 为什么做这个

过年、过节和朋友们打德州是固定节目，但经常遇到一个问题——**有牌、有桌、没筹码**。市面上的计分器要么只支持 8 人，要么不支持汇率换算，要么结算靠人工算，要么界面像 Excel。每次聚会都会说"下次一定做一个"，这次真的做了。

| 痛点 | 我们的方案 |
|------|-----------|
| 😤 只能 8 人？不够用 | ✅ **16 人上限**，大聚也能一桌搞定 |
| 💱 现金局要自己按汇率 ? | ✅ **内置汇率设置**，1 元 = X 积分，结算直接出金额 |
| 🧮 结算靠口算 / 计算器 | ✅ **最少转账次数算法**，自动生成最优转账清单 |
| 👀 UI 像临时凑的 | ✅ **暗色暖棕酒吧风**，WCAG AA 对比度，酒桌暗光也清晰 |
| 📵 退出忘记付钱 | ✅ **完整操作日志**，谁出了多少、谁收了池子全记录 |
| 🔄 多人同时操作乱套 | ✅ **乐观锁并发控制**，数据不乱不丢 |

## ✨ 功能一览

### 🏠 一键加入，无需注册
输入 4 位房间码即可加入，不注册、不登录，微信默认身份直接玩。

### ⚡ 毫秒级实时同步
基于 CloudBase 数据库 `watch()` API，任何玩家下注 / 收池，所有在线设备即刻更新。

### 🎯 操作极简，防误触设计
- 大按钮（≥48px），喝酒后手指不准也能按到
- 快捷下注 5/10/20/50/100/200 + ALL IN + 自定义
- ALL IN 二次确认，防止误触
- 下注后按钮 0.4 秒恢复，不卡顿

### 🧠 智能结算
最少转账次数算法，16 人场景下计算出最优转账方案，比来回转钱省事得多。

### ⚙️ 房主灵活设置
- 初始筹码（默认 1000）
- 单次下注上限（可选开启）
- 汇率（1 元 = X 积分）

### 📋 完整日志 & 分享
- 房间内小日志框实时滚动
- 点「查看全部」弹出完整操作记录，带时间戳
- 微信分享卡片，好友点击直接进房间

## 🏗️ 架构一览

```
                    微信小程序 (Client)
                   ┌─────────────────┐
                   │  pages/         │
                   │  ├── index      │  首页 — 创建/加入房间
                   │  ├── room       │  房间 — watch() 实时同步
                   │  ├── result     │  结算 — 转账清单
                   │  └── rules      │  牌型 — 10种牌型说明
                   │                 │
                   │  components/    │
                   │  ├── player-card│  玩家行组件
                   │  └── bet-picker │  下注选择器 (底部弹出)
                   └────────┬────────┘
                            │  wx.cloud.callFunction()
                            ▼
              ┌─────────────────────────┐
              │   微信云开发 CloudBase   │
              │                         │
              │  ☁️  云函数 (11个)       │
              │  ├── createRoom         │  生成4位房间码
              │  ├── joinRoom           │  加入房间 / 16人校验
              │  ├── bet                │  下注 / 乐观锁防并发
              │  ├── win                │  收池 / 乐观锁防并发
              │  ├── newGame            │  重置牌局
              │  ├── updateConfig       │  筹码设置 (房主)
              │  ├── settle             │  智能结算算法
              │  ├── leaveRoom          │  退出房间
              │  ├── transferOwnership  │  房主转让
              │  ├── updatePlayerProfile│  修改头像/昵称
              │  └── login              │  获取openid
              │                         │
              │  🗄️  数据库              │
              │  ├── Game  (房间文档)    │  ← watch() 实时监听
              │  └── Record (操作日志)   │
              └─────────────────────────┘
```

## 📂 项目结构

```
miniprogram/
├── app.js / app.json / app.wxss    入口文件 + 全局样式 (暖棕暗色主题)
├── pages/
│   ├── index/         首页 — 头像/昵称设置 + 创建/加入房间
│   ├── room/          房间主界面 — 三区固定布局 + watch() 实时同步
│   ├── result/        结算页 — 个人视图 + 全局转账清单
│   └── rules/         牌型说明 — 10种牌型从高到低排列
├── components/
│   ├── player-card/   玩家行 — 头像 + 昵称 + 投注 + 总分
│   └── bet-picker/    下注选择器 — 快捷金额 + ALL IN + 自定义
├── utils/
│   └── settle.js      最少转账次数结算算法 (云函数引用)
└── cloudfunctions/    11个云函数 (详见架构图)
```

## 🎨 设计

- **风格**：暖色酒吧 — 深棕底色 `#2c1810` + 暖橙点缀 `#e17055`
- **WCAG AA** 文字对比度 ≥ 4.5:1
- **三区固定布局**：顶部信息栏 + 中间玩家列表 + 底部操作区，拇指热区永远可见

## 🔧 本地开发

1. 微信开发者工具导入 `miniprogram/` 目录
2. 开通云开发，创建 `Game` 和 `Record` 数据库集合
3. 右键逐个上传部署 `cloudfunctions/` 下的 11 个云函数
4. 编译运行
